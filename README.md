# power_tool

电力系统工程近似计算工具。

## 1. 概述

本工具面向电力系统分析、设备参数折算、教学演示与方案前期校核场景。它不是潮流程序、机电暂态程序、EMT 平台或完整特征值分析平台，而是一个把“工程近似模型、参数换算、结果解释、图形展示”统一到同一桌面工具中的轻量级分析器。

---

## 2. 工具定位与适用范围

本工具服务于“快算、比选、复核、教学”四类场景。

第一，它适合在设计前期或方案讨论阶段快速判断量级，例如频率跌落、机电振荡频率、静态电压稳定极限、长线路自然功率、单机无穷大暂稳边界等。第二，它适合作为参数复核器，对架空线路、两绕组变压器、三绕组变压器进行有名值与标幺值换算，并给出常见越界提示。第三，它适合作为一个轻量化的分析器，例如对SMIB 模型进行平衡点构造、数值线性化、模态分析与参与因子解释。第四，它适合作为一个近似工程工具，对配电网多连接点合环工况做稳态环流、段电流分布与简化冲击暂态波形评估。

这一定义也意味着它有明确边界。它不替代全网潮流计算，不替代大型电力系统特征值分析平台，不替代正式的暂态稳定程序，不替代 PSCAD、EMTP、Simulink 等详细电磁暂态环境，更不替代保护整定和稳控定值审查。它的价值在于：用较小的输入规模，快速给出方向正确、量级可信、便于解释的工程结果。

## 3. 当前功能总览

当前主标签页共 10 个，另带一个固定在右侧的 `PowerTool AI` 问答侧栏：

1. 频率动态
2. 机电振荡
3. 静态电压稳定
4. 线路自然功率与无功
5. 暂稳评估
6. 小扰动分析（SMIB）
7. 配电网合环分析
8. 参数校核与标幺值
9. 短路电流计算
10. 录波曲线（COMTRADE）

`PowerTool AI` 侧栏不随主标签页和子标签页切换而移动，但软件界面中不再显示 AI 配置和标准提示词；相关内容统一写入 `power_tool_ai_config.json`。程序启动/提问时会读取该 JSON，在本地 `ollama` 与 API 模式间切换，默认使用本地 `qwen3.5:9b`。若使用 API 模式，配置文件中只保存环境变量名（例如 `DASHSCOPE_API_KEY`），程序会在运行时从环境变量读取密钥，从而避免把 key 明文写进配置文件。提问时程序会自动尝试截取当前软件界面，并把当前标签页算例数值摘要、标准系统提示词与用户问题一并发送给 AI。

其中，“参数校核与标幺值”内部又包含 3 个子页：“架空线路”“两绕组变压器”“三绕组变压器”；“小扰动分析（SMIB）”内部又包含 4 个子页：“工况与网络”“六阶机组”“AVR III”“PSS II”；“配电网合环分析”右侧绘图区还包含 2 个子页：“点位图与稳态电流”“冲击暂态电流”。

从问题域划分看，本工具可以分为四层：

- 近似解析层：频率动态、机电振荡、静态电压稳定、线路自然功率与无功。
- 稳定性层：暂稳评估、小扰动分析（SMIB）。
- 参数折算层：架空线路、两绕组变压器、三绕组变压器、线路几何反算。
- 故障与操作层：短路电流计算、配电网合环分析。

### 4.0 架构关系说明（重要）

PowerTool 本体仍然是独立软件：  
- 既可作为 GUI 桌面工具运行（`python power_tool.py`）；  
- 也可作为 Python 库调用（导入 `power_tool_*.py` 内核模块）。  

`skill` 与 `agent` 是包围在 PowerTool 外层的集成方式：  
- `skill` 负责把内核能力暴露为结构化调用接口；  
- `agent` 负责自然语言理解、工具编排和报告生成。  

因此两者与 PowerTool 理论上是**松耦合的外层软件**：  
- 不影响 GUI 主流程；  
- 可独立演进、替换模型或替换编排框架；  
- 在未启用 agent 时，PowerTool 仍可单独交付和运行。

为便于架构梳理，仓库新增 `PowerTool/` 目录作为“内层软件边界”标识，提供统一入口与核心清单；agent/skill 继续保留在外层。详见 `PowerTool/README.md`。
完整架构与测试说明见：`ARCHITECTURE.md`。

### 4.1 入口文件 `power_tool.py`

这是项目的直接启动入口。直接执行 `python power_tool.py` 会启动 GUI。同时它还承担兼容导出层的角色，尽量兼顾旧脚本的导入方式。

### 4.2 界面文件 `power_tool_gui.py`

该文件只处理 Tkinter 布局、Matplotlib 绘图、控件状态切换、文本结果组织和异常弹窗。工程上，这一层相当于“壳”，不再承担电力计算的核心逻辑。

### 4.3 公共基础 `power_tool_common.py`

这里统一放置输入校验函数、`InputError` 异常、JSON 读取函数和若干 dataclass 结果类型。所有内核模块都可以依赖它。

### 4.4 各物理问题的计算内核

- `power_tool_approximations.py`：频率动态、机电振荡、静态电压稳定、线路自然功率。
- `power_tool_stability.py`：冲击法、临界切除角、等面积法。
- `power_tool_smib.py`：SMIB 小扰动分析、特征值、参与因子。
- `power_tool_params.py`：线路与变压器的有名值/标幺值折算和参数校核。
- `power_tool_line_geometry.py`：由导线几何、土壤和地线数据反算序参数。
- `power_tool_faults.py`：故障类型、对称分量和短路电流计算。
- `power_tool_loop_closure.py`：多连接点合环稳态环流与 RL 暂态叠加。

## 5.6 Agent / Skill 无界面调用（本地 Ollama 或远程 API）

项目新增 `power_tool_skill.py` 与 `local_agent.py`：

- `power_tool_skill.py`：把各计算内核封装为统一技能入口 `execute_skill_request`，并提供技能目录 `list_skills`。
- `local_agent.py`：配置驱动的多步 Agent。可在 `local_agent_config.json` 中切换本地 `ollama` 或远程 OpenAI-compatible API，并设置推理步数、总时长、推理深度等约束。
- Agent 在最终响应时会自动生成一份 Markdown 汇总报告（`report_md`），包含问题、工具执行过程（逐步 JSON 结果）和结论，便于存档与复核。
- 若用户问题超出当前技能覆盖，Agent 会返回 best-effort 工程建议并明确局限，而不是直接失败退出。
- 技能层新增 `install_python_packages`，默认优先 `pandapower`，可用于复杂任务前的依赖准备（默认 dry-run，需显式允许才会安装）。
- 技能层新增 `pandapower_power_flow`，可在 Agent 中把 PowerTool 的近似/稳定分析与 pandapower 潮流计算结合使用，弥补“无系统潮流计算”的能力空缺。
- 技能层新增 `parse_pandapower_model`，可解析 pandapower JSON 模型文件，输出设备清单（bus/line/trafo/load/gen 等）和拓扑连接（edges + adjacency）。
- `local_agent_config.json` 支持 `bootstrap.install_pandapower_on_first_run=true`，可在首次启动 Agent 时自动安装 pandapower 并写入一次性标记文件。

快速示例：

```bash
python - <<'PY2'
from power_tool_skill import execute_skill_request

payload = {
    "skill": "frequency_dynamic",
    "args": {
        "delta_p_ol0": 0.08,
        "Ts": 8.0,
        "TG": 5.0,
        "kD": 1.2,
        "kG": 4.0,
        "f0_hz": 50.0,
    },
}
print(execute_skill_request(payload))
PY2
```

启动本地 Agent（需先启动 `ollama serve`）：

```bash
python local_agent.py
```

如果要改为远程 API 模型，可在 `local_agent_config.json` 中设置：

```json
{
  "provider": {
    "mode": "api",
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3.5-plus",
    "api_key_env": "DASHSCOPE_API_KEY"
  }
}
```

并在系统环境变量中设置对应 key（例如 `DASHSCOPE_API_KEY`）。

如果你希望“用户第一次用 PowerTool Agent 就自动装好 pandapower”，可以设置：

```json
{
  "bootstrap": {
    "install_pandapower_on_first_run": true,
    "marker_file": ".powertool_bootstrap_done.json"
  }
}
```

首次启动会自动执行安装；后续启动检测到 marker 文件后会跳过，避免重复安装。

如果要改用阿里源安装依赖（并且可随时切回默认源），可在同一配置文件设置：

```json
{
  "pip": {
    "index_mode": "aliyun",
    "index_url": "https://mirrors.aliyun.com/pypi/simple",
    "trusted_host": "mirrors.aliyun.com"
  }
}
```

切回官方默认源时，把 `index_mode` 改回 `"default"` 即可。若需企业内网源，使用 `"custom"` 并填写 `index_url`。

示例：先安装 pandapower（dry-run 改为实际安装时把 `allow_install` 设为 `true`），再执行潮流：

```bash
python - <<'PY2'
from power_tool_skill import execute_skill_request

print(execute_skill_request({"skill": "install_python_packages", "args": {"packages": ["pandapower"]}}))
print(
    execute_skill_request(
        {
            "skill": "pandapower_power_flow",
            "args": {
                "buses": [{"name": "BUS1", "vn_kv": 110.0}, {"name": "BUS2", "vn_kv": 110.0}],
                "ext_grid": {"bus": "BUS1", "vm_pu": 1.0},
                "lines": [{"from_bus": "BUS1", "to_bus": "BUS2", "r_ohm_per_km": 0.05, "x_ohm_per_km": 0.2, "length_km": 10.0}],
                "loads": [{"bus": "BUS2", "p_mw": 40.0, "q_mvar": 10.0}]
            }
        }
    )
)
PY2
```

如果用户提供 pandapower 模型文件（JSON），可先解析模型设备与拓扑：

```bash
python - <<'PY2'
from power_tool_skill import execute_skill_request

print(
    execute_skill_request(
        {
            "skill": "parse_pandapower_model",
            "args": {"model_path": "example_net.json"}
        }
    )
)
PY2
```

## 6. GUI 总体约定与阅读方式

这一节不是公式说明，而是界面使用说明。理解这些共性后，再看各标签页会更顺手。

### 6.1 左输入、右结果/图形是主布局

大多数主标签页采用“左侧输入与文本结果、右侧图形”的布局。没有独立图形的页签，会直接在下方给出只读文本结果框。

### 6.2 结果框全部是只读滚动文本框

所有“结果”区域都是 `ScrolledText`。计算后程序会重写内容，用户不需要也不应直接在结果框中编辑数字。其设计目标是：既可以显示结构化文字说明，也可以嵌入等宽字体的表格。

### 6.3 关键结论行会被红色高亮

如果结果文本中的某一行以前缀“运行区间判断：”“稳定性判断：”“结论：”“稳定性：”“匹配：”“不匹配：”开头，GUI 会自动把该行标成红色。这不是警报级错误，只是为了让使用者第一时间看到最重要的判断结论。

### 6.4 大多数页面启动后会自动算一遍默认值

当前版本中，频率动态、机电振荡、静态电压稳定、线路自然功率、暂稳评估、SMIB、线路参数校核、变压器参数校核、短路电流计算、配电网合环分析，都会在初始化阶段自动使用默认示例值运行一遍。这样打开程序后，用户可以先看默认结果，再开始修改。

### 6.5 图形下方都带 Matplotlib 工具栏

凡是有曲线图的页面，图下方都有 Matplotlib 导航工具栏，可用于平移、缩放、复位和保存图片。该工具栏属于通用 Matplotlib 组件，因此操作手势与科研/工程环境中的标准行为一致。

### 6.6 “框图”的含义

本 README 中提到“框图”，包含三类对象：

1. GUI 中的区块框，例如“冲击法快估”“等面积法（单机无穷大）”“合环近似参数”。
2. 图形类框，例如 P–δ 曲线图、短路波形图、合环点位图。
3. 传递函数结构图，例如 AVR III 和 PSS II 页面中的控制框图。

## 7. 主界面详细使用说明

这一节是本 README 的核心。下面按标签页顺序，逐页说明各个输入框区块、按钮、表格和图形的作用。

### 7.1 频率动态

#### 7.1.1 功能定位

该页用于计算含一次调频的二阶频率响应，并可同时叠加一个“无一次调频”的一阶对照曲线。它适合用来估算事故初期频率跌落、最低频率、RoCoF 和一次调频对 nadir 的影响。

#### 7.1.2 页面结构

左侧为输入区和结果区，右侧为频率曲线图。页面没有额外子页。

#### 7.1.3 输入框说明

| 输入框 | 含义 | 备注 |
|---|---|---|
| 额定频率 f0 / Hz | 系统基准频率 | 通常为 50 或 60 |
| 功率缺额 ΔP_OL0 / pu | 事故造成的有功失衡 | 按 pu 输入 |
| 系统惯性时间常数 T_s / s | 系统等效惯性 | 越大表示惯性越强 |
| 一次调频时间常数 T_G / s | 调速系统主时间常数 | 越大表示调频反应越慢 |
| 负荷频率系数 k_D / pu/pu | 负荷随频率变化的等效阻尼 | |
| 一次调频系数 k_G / pu/pu | 一次调频静态增益 | |
| 绘图时长 / s | 图形横轴终点 | 只影响绘图，不改变稳态结果 |

#### 7.1.4 复选框与按钮

- 复选框“同时绘制无一次调频一阶对照”：勾选后，右侧图中会同时画出一条只含负荷频率效应的一阶对照曲线，方便比较一次调频的作用。
- 按钮“计算并绘图”：读取全部输入，计算二阶模型关键指标，刷新结果框和曲线图。

#### 7.1.5 结果框内容

结果框会输出：阻尼类型、衰减系数 `α`、欠阻尼角频率 `Ω`（若存在）、RoCoF、稳态频差、最低频率时刻、最低频率值以及说明文字。

#### 7.1.6 右侧图形说明

右侧“频率曲线”会展示：

- 二阶模型的频率响应曲线；
- 若勾选复选框，则再叠加一阶对照曲线；
- 一条稳态频率水平线；
- 若存在 nadir，则在最低点位置打点标记。

这张图最适合回答两个问题：一是频率最低会跌到哪里，二是一次调频是否显著抬高最低频率。



### 7.2 机电振荡

#### 7.2.1 功能定位

|---|---|

##### 传递函数框图的功能说明

AVR 图是一个静态结构图，不参与数值求解本身，但它与实际求解模型一致。图中的信号流向是：

- 机端电压 `E_t` 进入测量环节 `1/(1+sT_r)`，输出 `v_m`；
- `v_m` 与参考电压 `V_ref`、PSS 输出 `V_s` 在求和点汇合；
- 求和后的信号依次通过 `K_0(1+sT_1)/(1+sT_2)` 和 `1/(1+sT_e)`；
- 输出为励磁量 `v_f`。

这个框图的意义在于：帮助使用者把“参数表”重新映射回实际控制结构，而不是仅仅面对一排数字。

#### 7.6.5 子页四：PSS II

这个子页包含 PSS 参数输入框和一张传递函数结构图。

##### 参数输入框

| 输入框 | 含义 |
|---|---|
| 洗出增益 Kw | PSS 主增益 |
| 洗出时间 Tw / s | 高通洗出环节时间常数 |
| 一阶超前 T1 / s | 第一组超前参数 |
| 一阶滞后 T2 / s | 第一组滞后参数 |
| 二阶超前 T3 / s | 第二组超前参数 |
| 二阶滞后 T4 / s | 第二组滞后参数 |
| 上限 vsmax / pu | PSS 输出上限 |
| 下限 vsmin / pu | PSS 输出下限 |

##### 传递函数框图的功能说明

PSS 图展示的是从转速偏差 `Δω` 到稳定器输出 `V_s` 的处理过程。信号依次通过：

1. 增益 `K_w`；
2. 洗出环节 `sT_w/(1+sT_w)`；
3. 第一组超前-滞后补偿 `(1+sT_1)/(1+sT_2)`；
4. 第二组超前-滞后补偿 `(1+sT_3)/(1+sT_4)`；
5. 最终输出 `V_s`，叠加到 AVR 求和点。

这张图的作用不是替代公式，而是让使用者更清楚地知道每个时间常数和增益在控制链中的位置。

结果框以等宽字体输出四类信息：

1. 配置类型、状态维数和稳定性判断；
2. 平衡点信息，包括 `V∞`、`Vt`、`P+jQ`、`δ0`、`pm0`、`vf0`、`id0`、`iq0`、`vd0`、`vq0` 等；
3. 最弱阻尼模态信息，包括该模态特征值、频率、阻尼比以及主导参与状态；
4. 模态表，只列出 `Im(λ) ≥ 0` 的独立模态。

其中“稳定性：”这一行会被红色高亮。

#### 7.6.7 右侧图：特征值复平面

复平面图以横轴 `Re(λ)`、纵轴 `Im(λ)` 的形式展示全部特征值。图中会自动在靠近虚轴或者最弱阻尼模态附近标记对应频率。它最适合回答两个问题：

- 系统是否稳定，也就是是否所有特征值都在左半平面；
- 哪个模态距离虚轴最近，也就是哪个模态阻尼最弱。

#### 7.7.2 说明文字区

“合环近似参数”框下面有一段固定说明，用来提示当前页面的录入约定：

- 每个连接点填写净线电流（A）；

#### 7.7.3 中部表格：连接点表

“连接点表”是本页最关键的可编辑表格。它会随着 `N` 自动重建。表头固定为 4 列：

| 列名 | 含义 |
|---|---|

#### 7.7.4 下部表格：线段比例（N+1 段）

由于 `N` 个连接点会把整条馈线划分成 `N+1` 个线段，因此本页单独提供“线段比例（N+1 段）”表。表头为“段号”和“比例”，每一段对应 `r1、r2、...、rN+1`。

它的作用不是直接输入 km 长度，而是输入相对比例。程序会自动按这些比例把“总线路长度 / km”分摊为各段长度，并写入右侧结果表与点位图。若所有比例都填 1，则表示均匀分布。

- 合环回路阻抗 `ZΣ` 及阻抗角；
- 稳态环流 `I_loop` 的极坐标表示；
- 暂态时间常数 `τ` 和 `2τ`；
- 左右源端合环前后电流变化；
- 环流和两侧总电流的瞬时峰值；

| 段号 | 从左到右的线段编号 |
| 区间 | 例如“左端->A”“B->联络点”“联络点->D” |
| 长度/km | 按比例分配后的线段长度 |
| 合环前/A | 该段合环前的电流幅值 |
| 合环后/A | 该段合环后的电流幅值 |
| 相角/° | 合环后电流相角 |
| 状态 | “正常”或“超限” |

这张表是工程判断的核心，因为它直接告诉使用者：哪一段是环流叠加后最危险的段，哪一段仍有余量。

#### 7.7.8 右下图页二：冲击暂态电流

#### 7.7.9 本页的工程边界

这一页的模型是近似工程工具，不是详细潮流 + EMT 联合仿真。连接点采用“净线电流”而不是完整负荷模型；暂态采用单一 RL 回路叠加，而不是多节点电磁暂态网络。因此它适合做工程快速判断，不适合替代 PSCAD、EMTP 或 Simulink 的精细开关暂态仿真。

### 7.10 录波曲线（COMTRADE）

#### 7.10.1 功能定位

该页用于加载 COMTRADE 录波文件（CFG + DAT），做多通道时域浏览、T1/T2 游标量测、窗口缩放、叠图查看，以及基于三相通道配置的序量分析。它适合做故障前后波形对比、相量/序量快速提取和录波教学演示。

#### 7.10.2 页面结构

左侧是文件与分析控制区，包括 CFG 选择、通道列表、时间窗设置、基波频率、三相电压/电流映射和序量结果文本；右侧是录波浏览区，显示当前时间窗内的多通道曲线及 T1/T2 游标。

#### 7.10.3 使用步骤

1. 点击“选择 CFG”并加载 COMTRADE 录波。
2. 在右侧主图中用左键放置 `T1`，用右键放置 `T2`。
3. 在左下角分别配置 `Ua/Ub/Uc` 与 `Ia/Ib/Ic` 对应的三相通道。
4. 点击“应用配置”，程序会按当前配置预计算并缓存整段录波上的序分量轨迹。
5. 之后无论是重新点击 `T1`，还是拖拽/平移/缩放主录波窗口，只要 `T1` 对应到的新时间点发生变化，左下角的序分量文本与相量图都会实时刷新。

#### 7.10.4 关于 T1 与拖拽联动的重要说明

这里的 `T1` 本质上是“当前可视窗口中的一个相对位置标记”。因此在屏幕上看起来 `T1` 竖线没有移动时，只要你拖拽了曲线、平移了时间窗，`T1` 实际对应的采样点和时间就已经变了。当前实现会在主图每次重绘后同步刷新序量分析区，确保左下角显示的 `V0/V1/V2/I0/I1/I2` 始终对应当前窗口下的真实 `T1` 时间。

#### 7.10.5 序量显示说明

序量相量图采用单坐标系综合显示，可同时绘制 `V0/V1/V2/I0/I1/I2` 六个向量。为保证拖拽时仍然流畅，序量分析会优先复用缓存好的序分量轨迹，而不是每次都对整段数据重新做一次相量提取。

## 10. 计算模型与物理假设摘要

这一节不追求推导完整，只归纳每个功能所依据的核心模型和应注意的简化假设。

### 10.1 频率动态

采用含一次调频的二阶频率响应近似模型，并可叠加一个无一次调频的一阶对照模型。适合系统级频率跌落量级估计，不适合多区域频率耦合和 AGC 细节分析。

### 10.2 机电振荡

采用经典功角模型下的同步转矩近似，得到主导机电振荡频率。适合快算，不等于全系统模态分析结果。

### 10.3 静态电压稳定

采用二节点、送端刚性、忽略电阻、负荷功率因数近似恒定的静稳极限公式。适合快算受端电压极限，不适合复杂负荷和多节点电压稳定研究。

### 10.4 线路自然功率与无功

基于波阻抗与自然功率概念估算长线路无功行为。适合超高压/高压长线路前期判断。

### 10.5 暂稳评估

- 冲击法：适合工程粗筛；
- 临界切除角快速估算：适合快速比对 `Δt` 与 `tcr`；
- 等面积法：适合单机无穷大经典模型的暂稳判据。

### 10.6 SMIB 小扰动分析

程序先根据运行点和网络参数构造平衡点，再对非线性模型数值线性化，最后求解特征值、阻尼比和参与因子。支持三种配置：六阶机组、六阶机组 + AVR、六阶机组 + AVR + PSS。

### 10.7 架空线路几何反算

采用三段完全换位平均；串联参数使用复深度近似；电容/电纳采用镜像法电位系数；地线启用时按连续接地导体并经 Kron 消去处理。它是工程级序参数反算器，不是多导体频变参数程序。

### 10.8 短路电流计算

采用对称分量和等值阻抗构造，计算常见故障类型的序电流和相电流，并用“交流分量 + 衰减直流分量”形式生成波形。适合工程快算，不代替保护级详细暂态程序。

### 10.9 配电网合环分析

采用“多连接点净线电流 + 两侧电压差 + 回路总阻抗”的近似模型。稳态部分先求合环环流，再叠加到左右两侧分段电流；暂态部分采用单一 RL 回路的合闸冲击叠加。适合做环流、电流分布和载流风险的快速判断。

## 12. 典型使用流程建议

### 12.1 做线路参数与短路联动

推荐流程是：

1. 在“架空线路”子页点击“线路参数计算”；
2. 在弹窗中录入导线几何、土壤和地线信息；
3. 点击“计算”确认 `Z1/Z0/C1/C0`；
4. 点击“回填正序到本页”，完成线路标幺值折算；
5. 点击“回填序参数到短路页”，把 `R1/X1/R0/X0` 直接送到故障计算页。

这样可以避免正序和零序参数在不同页面中手工重复输入。

### 12.2 做 SMIB 参数扫描

推荐流程是：

1. 点击“恢复 Kundur 默认值”；
2. 先在“六阶机组”模式下看基准模态；
3. 切换到“六阶机组 + AVR”，观察特征值是否向右移动；
4. 再切换到“六阶机组 + AVR + PSS”，观察最弱阻尼模态是否左移；
5. 配合 AVR/PSS 传递函数框图一起调整时间常数和增益。

### 12.3 做配电网合环快算

推荐流程是：

1. 先输入 `N`，再点击“按 N 重建表格”；
2. 在“连接点表”中录入各负荷点和 DG 点的净电流；
3. 选择一个净电流为 0 的空点作为合环点；
4. 录入 `U1/U2/φ/RΣ/XΣ` 以及总长度、载流量等参数；
5. 先看右侧结果框中的 `I_loop` 和“各段稳态电流表”；
6. 再看“点位图与稳态电流”确认危险区段；
7. 最后看“冲击暂态电流”判断瞬时峰值和暂态衰减是否可接受。

## 14. 适用边界与最终说明

需要再次强调：本工具的价值，不在于替代大型仿真软件，而在于把常用工程近似计算做成一个可解释、可复核、可视化、可扩展的小型平台。

如果把它当作“前处理器、教学器、快算器、参数复核器”，它是合适且高效的；如果把它当作“全网潮流、机电暂态、EMT、保护整定的一站式替代品”，那就超出了它的设计边界。

这种边界并不是缺点。相反，明确边界、在边界内做对，是工程软件最难也最重要的品质之一。
