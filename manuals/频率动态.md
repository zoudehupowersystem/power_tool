# 频率动态使用手册（工程版）

## 1. 功能定位与适用范围
频率动态页面用于事故后频率响应的快速工程估算，核心关注“最低频率、频率恢复速度、稳态偏差、一次调频效果”。
本页面适用于方案比选、调频参数整定前预估、运行人员培训演示，不替代全网机电暂态仿真。

## 2. 输入参数说明（建议先读）
- 额定频率 `f0`：通常 50 Hz。
- 功率缺额 `ΔP_OL0`：事故瞬间有功失衡，单位 pu。
- 系统惯性时间常数 `T_s`：反映惯性支撑能力，越大越“抗跌频”。
- 一次调频时间常数 `T_G`：反映调速系统响应速度，越小越快。
- 负荷频率系数 `k_D`：负荷对频率变化的自调节阻尼。
- 一次调频系数 `k_G`：调频力度增益。
- 绘图时长：建议覆盖“跌频—反弹—收敛”全过程。

## 3. 推荐操作流程
1. 先使用默认参数运行，观察曲线形态和结果区关键结论。
2. 固定 `ΔP_OL0`，分别调整 `T_s / T_G / k_G`，比较最低频率和恢复时间。
3. 若考虑 AGC，再增加绘图时长，重点观察中后期恢复趋势。
4. 形成“参数-结果”记录表，用于后续方案评审。

## 4. 结果解读方法
- 若最低频率过低：优先检查惯性（`T_s`）与调频速度（`T_G`）。
- 若恢复慢：通常与调频力度（`k_G`）或 AGC 参数不足有关。
- 若稳态偏差大：重点检查一次调频静态增益与负荷阻尼建模。

## 5. 常见问题与排查
- **问题1：曲线抖动或不平滑**
  - 检查输入是否极端（过大 `ΔP_OL0`、过小时间常数）。
- **问题2：结果看起来“过好”**
  - 核查参数单位是否统一为 pu/s/Hz。
- **问题3：与现场事件偏差较大**
  - 该页面是近似模型，未覆盖全网拓扑变化和保护动作逻辑。

## 6. 工程建议
- 将该页面作为“快速预审工具”，筛选明显不可行方案。
- 真正投运前仍需使用电网级动态仿真软件做最终验证。

## 附录A：参数核对清单
- 是否使用统一容量基准与电压基准。
- 是否确认输入量纲（pu、kV、MVA、Ω、秒）无误。
- 是否核查关键设备参数来源（铭牌/试验报告/调度台账）。
- 是否在高峰、平峰、低谷三个典型工况做过对比。
- 是否记录“输入参数—结论—建议动作”的闭环表格。

## 附录B：结果落地建议
1. 先把页面结果用于“快速筛选方案”。
2. 对筛选出的可行方案，建立复核算例（至少 3 组边界工况）。
3. 与保护、调度、运维联合审阅关键结论。
4. 形成可执行动作表：动作条件、执行步骤、回退条件、告警阈值。
5. 上线前在正式仿真平台做最终验证并归档。

## 附录C：常见问答（FAQ）
- **问：为什么页面结果与现场记录有偏差？**
  - 答：本软件是工程近似工具，未覆盖所有拓扑细节、控制死区和保护逻辑，建议将结果作为方向性参考。
- **问：如何提高结果可信度？**
  - 答：优先保证输入质量与基准一致性，再用多工况对比验证结论稳定性。
- **问：什么时候必须转正式仿真？**
  - 答：当涉及投运定值、保护整定、重大运行方式切换或安全边界接近时，应转入正式仿真流程。
